## 💡 Architecture Highlights

### ✅ Separation of Concerns
```
Input Layer (ui/)
    ↓
Data Layer (models/)
    ↓
Business Logic (engine/)
    ↓
Rendering Layer (renderers/)
    ↓
Output Layers (utils/)
```

### ✅ Single Design Object
All outputs derived from ONE unified `DesignObject`:
```python
DesignObject {
    incomers: [MCCB, MCCB, MCCB]
    outgoings: [MCCB, MCCB, MCCB]
    busbar: BusbarDesign
    controller: MicrogridController
    total_current_a: 825.5
}

# Used by:
- SLDRenderer → electrical diagram
- GARenderer → physical layout
- BOMGenerator → equipment list
- PDFReportGenerator → all 4 pages
```

### ✅ No Duplicate Logic
- Calculations only in `engine/calculations.py`
- Rendering only in `renderers/*.py`
- Constants only in `config/constants.py`

### ✅ Session State Management
```python
st.session_state["design"]        # DesignObject (cached)
st.session_state["sld_svg"]       # SVG (for preview)
st.session_state["sld_png_path"]  # Temp file (for PDF)

# Avoids recalculation on Download button
```

## 📊 Calculation Examples

### Electrical Calculations
```python
# Solar 100 kWp @ 415V, PF=0.8
Current = 100,000 / (√3 × 415 × 0.8) = 173.9A
MCCB = 173.9 × 1.25 = 217.4A → Select 250A ✓

# DG 250 kVA @ 415V, PF=1.0
Current = 250,000 / (√3 × 415) = 347.8A
MCCB = 347.8 × 1.25 = 434.75A → Select 500A ✓

# Busbar: Total ~825A (2-3 runs of 50×10mm Al)
```

All calculations are **pure Python functions** in `engine/calculations.py`
